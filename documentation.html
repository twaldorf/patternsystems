It's been almost a year since I started this tool. Amazing! I've made so little progress.

Anyway, a few things are on the immediate to-do:

    • And I also want to make posters for my room.

    Today I'm working on the graphics buffer implementation as this is the most fundamental problem.

    Currently, the program works like this: there is a render loop and if the cursor is over the vertex editor the loop runs. In the loop, the cursor is rendered, along with the all the shapes, directly on the canvas. When the user exports their pattern, a new graphics buffer is created to allow for the desired scaling and then the shape is rendered on the according to a new offset matrix that matches the parameters that the user set for their visible canvas.

    This sucks for a few reasons, but most importantly it sucks because what you see is not necessarily what you get and the shape rendering is duplicated in function although it's not identical due to the quirks of drawing in graphics buffers. Currently, because of the way colors were implemented, there is no fill or color in the exported images, rendering them literally unuseable.

    So the fix is to render each element to its own graphics buffer and then just re-render a version of the pattern buffer from the same parameters object. The cursor gets a buffer, the shape gets a buffer, and the pattern gets a buffer and the export function makes a new buffer that looks just like the pattern buffer but tileable.

    Once each element has its own buffer built from discrete parameter objects, adding support for multiple shapes and multiple tiling schemes will be trivial, which is a nice bonus.
